   - Rationale: pkg/fleetclient has no existing Ginkgo suite infrastructure, pure unit tests don't need BDD style
   - Impact: Simpler test file, follows Go standard library testing patterns

2. **PipelineID left optional (empty string = not available)**
   - Rationale: Client layer doesn't always have pipeline context (e.g., UpsertPipeline before ID assigned)
   - Impact: Controller in Phase 2 will populate PipelineID when wrapping/inspecting errors

3. **408 (Request Timeout) classified as transient**
   - Rationale: Industry standard retry guidance includes 408 as retriable
   - Impact: Broader retry classification, aligns with HTTP best practices

4. **No new dependencies added**
   - Rationale: All packages already in go.mod (Go 1.25, testify, net/http, errors)
   - Impact: Zero dependency churn, no security/compatibility risks
