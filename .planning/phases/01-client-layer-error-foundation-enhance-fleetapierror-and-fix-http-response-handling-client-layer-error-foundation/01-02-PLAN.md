Write the following test groups:

**1. TestFleetAPIError_IsTransient** (table-driven):
Test cases:
- 429 (Too Many Requests) -> true
- 408 (Request Timeout) -> true
- 500 (Internal Server Error) -> true
- 502 (Bad Gateway) -> true
- 503 (Service Unavailable) -> true
- 504 (Gateway Timeout) -> true
- 400 (Bad Request) -> false
- 401 (Unauthorized) -> false
- 403 (Forbidden) -> false
- 404 (Not Found) -> false
- 405 (Method Not Allowed) -> false
- 409 (Conflict) -> false
- 200 (OK, edge case) -> false

Use `http.StatusXxx` constants. For each case, construct `&FleetAPIError{StatusCode: code}` and assert `IsTransient()` equals expected.

**2. TestFleetAPIError_Error** (table-driven):
Test cases:
- With PipelineID: `FleetAPIError{StatusCode: 500, Operation: "UpsertPipeline", Message: "server error", PipelineID: "pipe-123"}` -> contains "pipeline=pipe-123", "status=500", "UpsertPipeline"
- Without PipelineID: `FleetAPIError{StatusCode: 400, Operation: "DeletePipeline", Message: "bad request"}` -> contains "status=400", "DeletePipeline", does NOT contain "pipeline="
- Empty message: verify it still formats correctly

**3. TestFleetAPIError_Unwrap**:
- Construct FleetAPIError with Wrapped set to a known error (e.g., `io.ErrUnexpectedEOF`)
- Assert Unwrap() returns the same error
- Construct FleetAPIError with nil Wrapped
- Assert Unwrap() returns nil

**4. TestFleetAPIError_ErrorsAs**:
- Wrap a FleetAPIError with fmt.Errorf("context: %w", apiErr)
- Use errors.As to extract it from the wrapped chain
- Assert extraction succeeds and fields match
- Double-wrap: fmt.Errorf("outer: %w", fmt.Errorf("inner: %w", apiErr))
- Assert errors.As still finds the FleetAPIError

**5. TestFleetAPIError_ErrorsIs**:
- Construct FleetAPIError with Wrapped = io.ErrUnexpectedEOF
- Assert errors.Is(apiErr, io.ErrUnexpectedEOF) returns true
- Wrap the FleetAPIError further: fmt.Errorf("wrap: %w", apiErr)
- Assert errors.Is(wrapped, io.ErrUnexpectedEOF) returns true (chain traversal)

Import `testing`, `errors`, `fmt`, `io`, `net/http`, and `github.com/stretchr/testify/assert`.

**6. TestUpsertPipeline_ReadBodyError** (TEST-01):
Use `net/http/httptest` to create a test server that:
- Returns a non-200 status code (e.g., 500)
- Sends partial/truncated response body by closing the connection mid-write

Approach: Create an httptest.Server with a handler that:
```go
handler := http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
    // Set status and headers
    w.Header().Set("Content-Length", "1000") // Claim large body
    w.WriteHeader(http.StatusInternalServerError)
    w.Write([]byte("partial")) // Write less than Content-Length
    // Hijack and close connection to force io.ReadAll error
    if hj, ok := w.(http.Hijacker); ok {
        conn, _, _ := hj.Hijack()
        conn.Close()
    }
})
```

Alternative simpler approach if Hijack is unreliable: Create a custom `io.Reader` wrapper. But since we're testing through the real HTTP client, use the httptest approach.

If the Hijack approach is flaky, use a simpler test: verify that when the server returns a non-200 status, the client returns a FleetAPIError with the body content (success path for io.ReadAll). Then separately test the error type construction is correct. The io.ReadAll error path is harder to trigger through HTTP but the code path is covered by the type tests.

**Practical alternative for io.ReadAll error testing:**
Since triggering io.ReadAll errors through httptest is inherently fragile, test it through a helper function approach:
- Extract the io.ReadAll + error handling into a testable helper, OR
- Simply test the HTTP client's error path for non-200 responses (which exercises the io.ReadAll success path) and rely on the FleetAPIError type tests for the error wrapping behavior.

**Recommended: Test the HTTP client error responses end-to-end:**

Test `UpsertPipeline` with httptest server returning various error codes:
- Server returns 400 with body "bad request" -> assert FleetAPIError with StatusCode=400, Message="bad request", Operation="UpsertPipeline"
- Server returns 500 with body "internal error" -> assert FleetAPIError with StatusCode=500, Operation="UpsertPipeline", IsTransient()=true
- Server returns 429 with body "rate limited" -> assert FleetAPIError with StatusCode=429, IsTransient()=true

Test `DeletePipeline` with httptest server returning error codes:
- Server returns 500 with body "server error" -> assert FleetAPIError with StatusCode=500, Operation="DeletePipeline", PipelineID is set
- Server returns 404 -> assert NO error (404 = success for delete)
- Server returns 200 -> assert NO error

For each test:
1. Create httptest.NewServer with appropriate handler
2. Create client with NewClient(server.URL+"/", "user", "pass")
3. Call the method
4. Assert error type and fields using errors.As

**Important implementation notes:**
- The client prepends the operation name to baseURL (e.g., baseURL+"UpsertPipeline"), so the test server URL should end with "/" and the handler should not care about the path
- The client uses Basic Auth, the test server handler can ignore it
- The client expects JSON for successful UpsertPipeline responses, but error paths don't parse JSON
- Use `defer server.Close()` in each test
- Use `context.Background()` for the context
