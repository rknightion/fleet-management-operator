  <files>test/mockapi/main.go, test/mockapi/Dockerfile</files>
  <action>
Create a minimal Go HTTP server in test/mockapi/main.go that mimics the Fleet Management Pipeline API.

The mock server must implement these endpoints matching the real API contract in pkg/fleetclient/client.go:

1. POST /pipeline.v1.PipelineService/UpsertPipeline
   - Accept JSON body: {"pipeline": {"name": "...", "contents": "...", "matchers": [...], "enabled": true/false, "configType": "...", "source": {...}}, "validateOnly": false}
   - Store pipeline in memory (sync.Map keyed by name)
   - Generate a unique ID (use atomic counter starting at 1000, format as string)
   - Return JSON: {"name": "...", "contents": "...", "matchers": [...], "enabled": ..., "id": "...", "configType": "...", "createdAt": "...", "updatedAt": "..."}
   - Timestamps should be RFC3339 format (time.Now().UTC().Format(time.RFC3339))
   - If pipeline already exists by name, reuse its ID (upsert semantics)
   - Handle basic auth header (accept any username/password, just verify header exists)
   - Return 401 if no basic auth header

2. POST /pipeline.v1.PipelineService/DeletePipeline
   - Accept JSON body: {"id": "..."}
   - Remove from memory store
   - Return 200 with {"status": "ok"} (even if not found, for idempotent delete behavior)
   - Handle basic auth header same as above

The server should:
- Listen on port 8080 (configurable via PORT env var)
- Log all requests to stdout for debugging
- Use standard library only (net/http, encoding/json, sync, sync/atomic)
- NOT use any external dependencies (keep it simple for a test utility)
- Include a GET /healthz endpoint returning 200 for readiness probes

Create test/mockapi/Dockerfile:
- Use golang:1.25 as builder stage
- Copy only test/mockapi/ directory
- Build the binary with CGO_ENABLED=0
- Use gcr.io/distroless/static:nonroot as runtime
- Expose port 8080
- IMPORTANT: This is a standalone Go binary, NOT a module. Use a minimal go.mod with just "module mockapi" and "go 1.25.0". Do NOT import any project packages.
  </action>
