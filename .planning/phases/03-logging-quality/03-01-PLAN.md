In `internal/controller/errors.go`, add two new functions:

1. **formatConditionMessage(reason string, err error) string** - Maps errors to user-friendly status condition messages with troubleshooting hints. Uses errors.As to check for FleetAPIError and switches on StatusCode:
   - 400 BadRequest: "Configuration validation failed: {message}. Review spec.contents for syntax errors."
   - 401 Unauthorized: "Authentication failed. Verify Fleet Management credentials in the referenced Secret."
   - 403 Forbidden: "Permission denied. Fleet Management access token requires pipeline:write permission."
   - 404 NotFound: "Pipeline not found in Fleet Management. It may have been deleted externally."
   - 429 TooManyRequests: "Rate limited by Fleet Management API. Retry will occur automatically after delay."
   - 500/502/503: "Fleet Management API unavailable (HTTP {code}). Retry will occur automatically."
   - default: "API error (HTTP {code}): {message}"
   - context.DeadlineExceeded: "Connection timeout. Check network connectivity to Fleet Management API."
   - net.Error timeout: "Network timeout. Check Fleet Management API endpoint is reachable."
   - generic fallback: "Sync failed: {err}. Check controller logs for details."

   Add imports: "context", "fmt", "net/http". Note: net import already exists.

2. **loggerFor(ctx context.Context, pipeline) logr.Logger** - This is a standalone function (not a method on PipelineReconciler) that wraps logf.FromContext(ctx) with pipeline namespace and name:
   ```go
   func loggerFor(ctx context.Context, ns, name string) logr.Logger {
       return logf.FromContext(ctx).WithValues(
           "namespace", ns,
           "name", name,
       )
   }
   ```
   Add imports: "github.com/go-logr/logr", logf "sigs.k8s.io/controller-runtime/pkg/log"

In `internal/controller/pipeline_controller.go`, add condition state transition logging:
