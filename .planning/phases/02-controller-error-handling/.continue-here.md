<context>

**What was accomplished:**
Phase 2 fixed three critical controller bugs that could cause production failures:
1. Status update errors replacing original errors (breaking exponential backoff)
2. Infinite recursion in 404 external deletion handling (stack overflow risk)
3. No distinction between transient and permanent errors (improper retry behavior)

The error classification helpers (isTransientError, shouldRetry) provide centralized retry logic that integrates with FleetAPIError.IsTransient() from Phase 1. The updateStatusError fix ensures controller-runtime always sees the original reconciliation error for proper exponential backoff. The 404 inline recreation pattern prevents infinite recursion with a simple Status.ID guard.
