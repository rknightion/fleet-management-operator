Two fixes in `internal/controller/pipeline_controller.go`:

**Fix 1: updateStatusError (lines 362-405) -- preserve original error**

The current bug: when `r.Status().Update()` fails with a non-conflict error (line 394), it returns `updateErr` instead of the original `err`. This breaks exponential backoff because controller-runtime sees the status update error, not the reconciliation error.

Replace the updateStatusError function body with:
- Keep the existing status condition setup (lines 367-385) exactly as-is (ObservedGeneration, Ready condition, Synced condition).
- Change the status update error handling (lines 388-395):
  - On conflict (`apierrors.IsConflict(updateErr)`): Keep returning `ctrl.Result{Requeue: true}, nil` (correct behavior, cache is stale).
  - On other status update errors: LOG the status update error with full context (include `originalError` as a structured key-value pair with `err.Error()`, include `reason`), but do NOT return it. Fall through to the original error return logic.
- Keep the validation error check (lines 398-401): `if reason == reasonValidationError` return `ctrl.Result{}, nil`.
- Final return: `return ctrl.Result{}, err` (the ORIGINAL error, for exponential backoff).

The critical change is line 394: change `return ctrl.Result{}, updateErr` to just logging updateErr and falling through. The function parameter name for the error is already `err` -- keep it as `err` for consistency with the existing signature.

**Fix 2: handleAPIError StatusNotFound case (lines 263-269) -- prevent infinite recursion**

Current bug: line 269 calls `r.reconcileNormal(ctx, pipeline)` after clearing pipeline.Status.ID. If the recreation also fails with 404, this recurses infinitely.

Replace the `case http.StatusNotFound:` block (lines 263-269) with:
1. Check if `pipeline.Status.ID == ""` (already tried recreation):
   - If yes: log error, emit Warning event with `eventReasonSyncFailed` message "Failed to recreate pipeline after external deletion", return `r.updateStatusError(ctx, pipeline, reasonSyncFailed, fmt.Errorf("pipeline not found and recreation failed: %w", err))`.
2. If `pipeline.Status.ID != ""` (first detection of external deletion):
   - Log with `"previousID"` key showing the old ID.
   - Emit Warning event with `eventReasonRecreated` message "Pipeline was deleted externally, recreating in Fleet Management".
   - Clear `pipeline.Status.ID = ""`.
   - Call `r.buildUpsertRequest(pipeline)` to get a fresh request.
   - Call `r.FleetClient.UpsertPipeline(ctx, req)` INLINE (no recursion).
   - If UpsertPipeline fails: call `r.handleAPIError(ctx, pipeline, err)` to classify the NEW error. This is safe because pipeline.Status.ID is now empty, so if it hits 404 again it will take the "already tried" branch above.
   - If UpsertPipeline succeeds: call `r.updateStatusSuccess(ctx, pipeline, apiPipeline)`.

Also update the type assertion on line 254 from `err.(*fleetclient.FleetAPIError)` to use `errors.As` for consistency with the new errors.go pattern. Change:
```go
if apiErr, ok := err.(*fleetclient.FleetAPIError); ok {
```
to:
```go
var apiErr *fleetclient.FleetAPIError
if errors.As(err, &apiErr) {
```
This requires adding `"errors"` to the imports.

Add `"errors"` to the import block if not already present.
