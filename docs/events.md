# Event Reference

Each controller emits Kubernetes events on significant reconcile outcomes.
Events are user-facing log text — surface them via:

```
kubectl describe <kind> <name>
kubectl get events --field-selector involvedObject.kind=<Kind>
```

Reasons are stable PascalCase identifiers and may be matched on directly. Event
type `Warning` indicates the controller could not complete its work; `Normal`
indicates a routine state transition.

For status-condition reasons (a separate, also-stable surface), see
[conditions.md](conditions.md).
